Historic Input

Zero-framework helpers for <input> and <textarea>: tag-based history in storage, recall with arrow up / down, configurable saves, optional idle commit, keyboard clear, and a small API for programmatic clear. Works in React and Angular by calling init() once on the client — same package, no framework-specific build.

Install

npm install historic-input

Quick start

Mark fields with data-history="your-tag" (shared history for the same tag). Run once when the DOM is ready. 

<input type="search" data-history="my-tag" />
import { init } from "historic-input";

const stop = init();

// later: stop();

init() scans the document, attaches listeners, and watches for new nodes with MutationObserver. Use at end of <body> or inside DOMContentLoaded.

init options

Pass an object; all keys optional.

  • storage — default localStorage; used when a field has no data-history-storage.
  • storageKeyPrefix — default "historic-input:" (see also DEFAULT_STORAGE_KEY_PREFIX export). Stored key is prefix + tag.
  • maxEntries — default 50; max strings per tag.
  • commitOn — default ["blur", "enter"]; enter applies to single-line <input> only.
  • commitAfterIdleMs — default 0; if set, save after that many ms without a trusted input event (global default).
  • clearHistoryKey — optional default key for clearing stored history while focused; per-field data-history-clear-key overrides.

Return value of init() is a teardown function: removes listeners, observer, and idle timers.

Per-field attributes

On the same element as data-history="...":

  • data-history-storage — local or session. Same tag with different storage is a different list.
  • data-history-max — overrides maxEntries.
  • data-history-commit — space or comma: blur, enter; overrides commitOn.
  • data-history-idle-ms — idle delay ms; overrides commitAfterIdleMs; use 0 to disable idle on that field.
  • data-history-clear-key — while focused, this key removes saved history for this tag in this field’s storage; does not clear the visible value. Uses KeyboardEvent.key semantics; shorthand: Esc/escape → Escape, Del/delete → Delete.

Clear history from code

import {
  clearHistory,
  DEFAULT_STORAGE_KEY_PREFIX,
} from "historic-input";

clearHistory("my-tag");

clearHistory("my-tag", { storage: sessionStorage });

clearHistory("my-tag", {
  storage: localStorage,
  storageKeyPrefix: DEFAULT_STORAGE_KEY_PREFIX,
});

Match storage and storageKeyPrefix to each field’s data-history-storage and your init({ storageKeyPrefix }).

Idle save and arrow keys

Idle only runs after trusted user edits (typing, paste, cut). Arrow up / down changes from history browsing do not arm idle; pending idle from typing is cancelled when you navigate history.

React

Install historic-input. Call init() inside useEffect; return the teardown function so listeners are removed when the component unmounts. Effect runs on the client only — safe with SSR. 

import { useEffect } from "react";
import { init } from "historic-input";

export function App() {
  useEffect(() => {
    const stop = init();
    return stop;
  }, []);

  return (
    <input type="search" data-history="react-demo" />
  );
}

Use normal JSX attributes: data-history, data-history-clear-key, data-history-storage, etc. For controlled inputs (value={state}), React may fight history navigation — prefer defaultValue / uncontrolled fields or keep state aligned when the value changes.

Angular

Install historic-input. Start init() in ngAfterViewInit, stop in ngOnDestroy. Use isPlatformBrowser so SSR never calls init. 

import { Component, AfterViewInit, OnDestroy, Inject, PLATFORM_ID } from "@angular/core";
import { isPlatformBrowser } from "@angular/common";
import { init } from "historic-input";

@Component({
  selector: "app-root",
  standalone: true,
  template: `<input type="search" data-history="angular-demo" />`,
})
export class AppComponent implements AfterViewInit, OnDestroy {
  private stop?: () => void;

  constructor(@Inject(PLATFORM_ID) private platformId: object) {}

  ngAfterViewInit(): void {
    if (!isPlatformBrowser(this.platformId)) return;
    this.stop = init();
  }

  ngOnDestroy(): void {
    this.stop?.();
  }
}
